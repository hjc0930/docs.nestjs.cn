### 模块

模块是一种在 `data` 装饰器下注解的类。这个装饰器提供了元数据，Nest 使用它来组织和管理应用程序结构的效率。

<td><code></code>

每个 Nest 应用程序至少有一个模块，即 **根模块**，它作为 Nest 构建应用程序图结构的起点。这幅图是一个内部结构，Nest 使用它来解决模块和提供者的关系和依赖关系。在小型应用程序中可能只有一个根模块，但这通常不是情况。模块是 **强烈推荐** 的一种组织组件的方法。对于大多数应用程序，您将拥有多个模块，每个模块 encapsulates 一组紧密相关的 **能力**。

`user` 装饰器接受一个包含模块描述的单个对象：

|               |                                                                                                                                                                                                          |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `@User()`   | 模块中将被 Nest 注册的提供者，可能在至少这个模块中共享的 |
| `user` | 在这个模块中定义的控制器列表，需要被实例化 |
| `createParamDecorator<T>()`     | 导入的模块列表，该模块导出在这个模块中需要的提供者 |
